# 📋 Чек-лист UI автотестов: Way2Automation

## 1. Главная страница (Way2Automation)
- [ ] **1.1 Проверка открытия страницы**
    - Отображение хедера (контакты), блока навигации, кнопки регистрации.
    - Отображение списка курсов (Best Selenium Certification).
    - Отображение футера.
- [ ] **1.2 Проверка хедера (Contact Info)**
    - Наличие номеров телефонов, Skype, e-mail и иконок соцсетей.
- [ ] **1.3 Слайдер курсов (Most Popular)**
    - Проверка кнопок «Вперед» и «Назад»: корректная смена слайдов.
- [ ] **1.4 Футер**
    - Наличие актуального адреса, контактов и почты.

## 2. Навигация и Скроллинг
- [ ] **2.1 Sticky Menu (Липкое меню)**
    - Меню остается видимым в верхней части экрана при скролле вниз.
- [ ] **2.2 Переходы по меню**
    - Переход: *All courses -> Lifetime membership*.
    - Проверка URL: `.../lifetime-membership-club/`.
    - Проверка заголовка: "LIFETIME MEMBERSHIP CLUB".

## 3. Авторизация (Protractor Practice)
- [ ] **3.1 Проверка полей ввода**
    - Отображение полей Username и Password.
    - Кнопка Login заблокирована, если поля пустые.
- [ ] **3.2 Успешный вход**
    - Ввод валидных данных -> Сообщение "You're logged in!!".
- [ ] **3.3 Ошибка авторизации**
    - Ввод невалидных данных -> Сообщение "Username or password is incorrect".
- [ ] **3.4 Разлогирование**
    - Нажатие Logout -> Возврат к форме ввода (поля отображаются).

## 4. Банковское приложение (XYZ Bank)

### 4.1 Регистрация (Sample Form)
- [ ] Заполнение формы корректными данными.
- [ ] Выбор хобби (Sports).
- [ ] Динамическое вычисление самого длинного слова в поле "About Yourself".
- [ ] Проверка сообщения "User registered successfully!".

### 4.2 Менеджер банка (Bank Manager)
- [ ] **Добавление клиента**: Заполнение данных -> Проверка Alert-окна.
- [ ] **Открытие счета**: Выбор клиента и валюты -> Проверка Alert-окна.
- [ ] **Удаление клиента**: Поиск по имени в списке -> Нажатие Delete -> Проверка отсутствия.

### 4.3 Клиент (Customer Login)
- [ ] **Вход**: Приветствие "Welcome [Имя] !!".
- [ ] **Депозит (Успех)**: Сумма 100321 -> Сообщение "Deposit Successful" -> Проверка в Transactions.
- [ ] **Депозит (Ошибка)**: Сумма 0 -> Отсутствие сообщения об успехе и записи в транзакциях.
- [ ] **Снятие (Успех)**: Снятие суммы N (от 1 до баланса) -> "Transaction successful" -> Проверка в Transactions.
- [ ] **Снятие (Ошибка)**: Сумма > баланса -> Сообщение "Transaction Failed...".
- [ ] **Проверка баланса**: Сверка значения на главной со суммой всех строк в Transactions.
- [ ] **Обнуление**: Снятие остатка под 0 -> Проверка "Balance : 0".
- [ ] **Очистка**: Кнопка Reset в транзакциях -> Проверка пустоты таблицы и баланса.
