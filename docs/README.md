## 📋 Описание проекта

Профессиональный фреймворк для автоматизированного тестирования веб-интерфейсов. Построен на базе паттерна **Page Object Model (POM)** с фокусом на стабильность и читаемость тестов.

## 🛠 Технологический стек
- **Python 3.10+**
- **Pytest** — основной фреймворк и тест-раннер.
- **Selenium WebDriver** — управление браузером.
- **Allure Report** — интерактивная отчетность с шагами, скриншотами и логами.
- **Pydantic Settings** — типизированное управление конфигурацией через `.env`.
- **Pytest-xdist** — поддержка параллельного запуска тестов.

## 📁 Структура проекта
- `base/` — Ядро фреймворка: базовый класс `BasePage` с безопасными обертками над Selenium (Safe Clicks, Elements Search).
- `config/` — Управление окружением и глобальными таймаутами.
- `pages/` — Описание страниц: разделение локаторов и бизнес-логики взаимодействия.
- `tests/` — Тестовые сценарии (Smoke, Regression, UI-widgets).
- `utils/` — Вспомогательные инструменты, включая `wait_to_change` (Smart Polling для асинхронных элементов).

## 🚀 Быстрый старт

### 1. Подготовка окружения
```bash
# Создание и активация виртуального окружения
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
.venv\Scripts\activate     # Windows

# Установка зависимостей
pip install -r requirements.txt
```

### 2. Настройка конфигурации
Создайте файл `.env` в корне проекта на основе `config/pydantic_config.py`.
```env
BASE_URL="https://www.way2automation.com"
DEFAULT_TIME_OUT=15
POLL_FREQUENCY=1
HEADLESS=true
```
Настройка тестовых данных

1. Скопируйте `test_data_example.xlsx` в `test_data.xlsx`
2. Заполните файл тестовыми данными
3. Файл `test_data.xlsx` добавлен в `.gitignore` и не будет пушиться в репозиторий

### 3. Запуск тестов
```bash
pytest ./tests/ -n 3 --alluredir=allure-results -v
```

## 📝 Особенности реализации
- **Smart Waiters:** Вместо нестабильных `time.sleep` используется кастомная утилита `wait_to_change`. Она позволяет дождаться изменения текста или состояния элемента (например, переключения слайда), что критично для динамических UI-компонентов.
- **Headless Stability:** Для корректного отображения элементов в Headless-режиме принудительно задается разрешение **1920x1080** и валидный **User-Agent**. Это предотвращает падения из-за перестроения сайта в мобильную верстку.
- **Allure Steps:** Каждый метод в `BasePage` обернут в `with allure.step`, что делает отчеты максимально детализированными без дублирования кода в тестах.
